# 20 ВОПРОСОВ НА СЕНЬОРНОСТЬ

**34.** Резолвер pip (`resolvelib`) реализует CDCL-подобный backtracking, но без conflict-driven clause learning. Sat-резолверы (Dart pub, PubGrub) учат «инкомпатибельности» и не повторяют ошибку. Приведи граф, на котором pip перебирает экспоненциально, а PubGrub заканчивает за линейное число шагов. Почему pip до сих пор не перешёл?

**35.** У pip нет доступа к метаданным sdist без сборки: чтобы узнать `install_requires` у `foo==1.2.3`, он скачивает архив и выполняет `setup.py egg_info` — то есть **выполняет произвольный код автора пакета при резолвинге, даже если пакет в итоге не будет установлен**. PEP 658 (`.metadata` рядом с wheel) закрыл это частично. Что осталось открытым и почему это остаётся supply-chain дырой?

**36.** Poetry по умолчанию генерирует кросс-платформенный lock: одна запись покрывает все ОС/версии Python через маркеры. `uv` до 0.4 требовал `--universal` явно. Почему кросс-платформенный lock **математически не всегда возможен**? Приведи пример, где корректного универсального решения не существует.

**37.** `^1.2.3` в Poetry ≡ `>=1.2.3,<2.0.0`, но `^0.2.3` ≡ `>=0.2.3,<0.3.0`. Это правило пришло из npm. Почему для `0.x` семантика ломается на практике и какую конкретную боль это создаёт в экосистеме, где половина ML-библиотек годами живёт на `0.x`?

**38.** PEP 440 нормализация: `1.0.0-alpha1`, `1.0.0a1`, `1.0.0.alpha.1`, `1.0-a1` — одна и та же версия. PEP 503 нормализация имён: `Foo.Bar_baz` → `foo-bar-baz`. Назови сценарий, где различие «сырого» и нормализованного имени приводит к тому, что два **разных** дистрибутива устанавливаются в одну директорию и молча перетирают друг друга.

**39.** Python не поддерживает две версии одного пакета в одном процессе — `sys.modules` ключуется по имени. Node решает через nested `node_modules`, Rust через семверную реномнацию символов в crate metadata, Go через `/v2` в import path. Почему для Python ни один из трёх подходов не был принят? Что конкретно ломается (назови механизм импорта)?

**40.** У тебя монорепо: 40 сервисов, общий lock. `pip install` холодный — 4 минуты, `uv sync` — 6 секунд. Разложи разницу по компонентам: сколько даёт HTTP-кеш метаданных, сколько — параллельная загрузка, сколько — hardlink вместо copy в site-packages, сколько — сам резолвер. Какой компонент нельзя перенести в pip без изменения PEP?

**41.** `uv` устанавливает пакеты hardlink-ами из глобального кеша в venv. Что произойдёт, если процесс откроет файл из site-packages на запись? Какая изоляция венвов при этом нарушается и почему `uv` считает это приемлемым trade-off?

**42.** Dependency confusion (Alex Birsan, 2021): пакет `internal-utils` есть в приватном индексе версии `1.0.0`, атакующий публикует `internal-utils==99.0.0` на PyPI. При `--extra-index-url` pip берёт 99.0.0. Почему `--index-url` + `--find-links` не полное решение? Что предлагает PEP 708 и почему он не решает задачу целиком?

**43.** Проект зависит от `numpy>=1.20` и от `pandas`, который зависит от `numpy>=1.23`. Резолвер выдаёт `numpy==2.1`. Всё ставится, `pip check` чист, импорт падает с `ValueError: numpy.dtype size changed, may indicate binary incompatibility`. Объясни механизм. Почему система версий Python принципиально не видит эту несовместимость?

**44.** `setup.py` vs `pyproject.toml`: до PEP 621 метаданные были императивными — версия могла зависеть от `git describe`, зависимости от `sys.platform` внутри Python-кода. PEP 621 сделал их декларативными. Что мы потеряли на этом переходе? Назови кейс, который честно решался в `setup.py` и не решается в статическом `[project]`.

**45.** `pip install -r requirements.txt` не удаляет пакеты, которых нет в файле. `poetry install --sync` и `uv sync` удаляют. Почему pip до сих пор не имеет `--sync` по умолчанию и какой класс production-инцидентов создаёт именно накопительная семантика pip?

**46.** У тебя 500 микросервисов, общая внутренняя библиотека `core`, релиз раз в неделю. Вариант A: все пиннят `core==X.Y.Z`, апгрейд PR-ом в каждый репо. Вариант B: все на `core>=X,<X+1`, апгрейд при следующем деплое. Вариант C: монорепо + один lock. Разбери каждый по осям: время до фикса CVE, риск разъезда, стоимость CI, blast radius плохого релиза. Что выберешь и при каком размере команды ответ меняется?

**47.** Hash-pinning (`--require-hashes`) фиксирует sha256 каждого артефакта. Атакующий скомпрометировал не PyPI, а CI автора пакета — новая версия выпущена с бэкдором и **правильным** хешем. Что здесь ломается? Как соотносятся PEP 458/480 (TUF) и Sigstore-аттестации по модели угроз, и почему TUF на PyPI буксует с 2013 года?

**48.** `pip install foo` при наличии установленного `foo` с другой версией: pip удаляет старую, ставит новую — **не атомарно**. Ctrl+C посередине → окружение в промежуточном состоянии, `.dist-info` есть, файлов нет. Как реализованы транзакционные установки в `uv`? Почему полная атомарность на POSIX всё равно недостижима (назови конкретный шаг)?

**49.** Тесты падают в CI, локально зелёные. Lock-файл один и тот же. Назови пять механизмов, которыми окружения могут разъехаться при идентичном lock. (подсказка не нужна — их больше пяти)

**50.** `pip install` резолвит по одному пакету за раз и при конфликте бэктрекается, скачивая всё новые sdist. На пакете вроде `apache-airflow` (600+ транзитивных зависимостей) это заканчивалось часами. Airflow ответил официальным constraints-файлом на каждую версию. Это признание какого именно провала — резолвера, экосистемы или модели версионирования? Аргументируй.

**51.** Extras (`pip install foo[redis]`) не участвуют в резолвинге как отдельные узлы графа до PEP 685 нормализации. Приведи сценарий, где `foo[a]` и `foo[b]`, затянутые двумя разными пакетами, дают разный набор версий в зависимости от **порядка** их появления в requirements. Это баг или спецификация?

**52.** Rust/Cargo: два минорных диапазона одного crate сосуществуют в одном бинарнике, конфликт версий как класс отсутствует. Go: MVS (minimal version selection) — выбирается **минимальная** удовлетворяющая версия, резолвинг детерминирован и линеен, без backtracking. Почему MVS невозможно применить к PyPI как есть? Что конкретно в модели Python/PyPI это блокирует?

**53.** Ты проектируешь резолвер для Python с нуля в 2026: без legacy `setup.py`, но с обязательной обратной совместимостью по установке существующих 600k пакетов PyPI. Три решения, которые ты принимаешь в первую очередь, и цена каждого. Где ты сознательно ломаешь совместимость?
