# 20 ВОПРОСОВ НА СЕНЬОРНОСТЬ

**38.** `do_richcompare` в `Objects/object.c` проверяет `PyType_IsSubtype(Py_TYPE(w), Py_TYPE(v))` перед первым вызовом. Какой конкретный сценарий с подклассом эта проверка чинит и что бы сломалось без неё?

**39.** До Python 3.0 существовал `__cmp__`. Какой PEP его убил, чем rich comparison лучше и какую задачу `__cmp__` решал элегантнее? Что пришлось вернуть в виде `cmp_to_key`?

**40.** Класс с `__eq__`, наследующийся от `int`, положен в `set` вместе с обычными int. Хеши совпадают, `__eq__` асимметричен. Опиши, при каком порядке вставки `len(set)` окажется 1, а при каком — 2. Почему `set` не гарантирует детерминизм здесь?

**41.** `hash(x) == hash(y)` при `x != y` — коллизия. В CPython dict это `open addressing` с `perturb`-пробированием. Сколько бит хеша использует первая проба, откуда берутся остальные и почему NaN-подобный класс с `__hash__ = lambda self: 0` даёт O(n²) на вставке n элементов?

**42.** PEP 456 (SipHash) — от какой атаки он защищает и почему рандомизация не распространяется на `hash(int)`? Можно ли построить DoS на dict через int-ключи и что этому мешает?

**43.** `functools.total_ordering` генерирует `__gt__` из `__lt__` как `not (self < other or self == other)`. Для какого домена данных эта формула даёт неверный ответ и почему стандарт IEEE 754 делает её принципиально нечинибельной?

**44.** `sorted(lst)` на списке из 1M float без NaN и с одним NaN в середине. Первый случай — корректно, второй — молча выдаёт частично отсортированный список без исключения. Почему Timsort не падает, а именно молча ломается? Где именно теряется инвариант galloping-режима?

**45.** В Java `Comparable` контракт требует `sgn(x.compareTo(y)) == -sgn(y.compareTo(x))`, и `TreeMap` может выкинуть исключение при нарушении. В Rust есть `Ord` vs `PartialOrd` — разные трейты, и float реализует только второй. Python не различает их вообще. Какую цену Python платит за это и что бы дал раскол `__lt__` на total/partial?

**46.** `dict` lookup делает `PyObject_RichCompareBool(startkey, key, Py_EQ)`, который начинается с `if (v == w) { if (op == Py_EQ) return 1; }`. Приведи класс, где эта оптимизация даёт наблюдаемо неверный ответ по сравнению с прямым вызовом `==`. Это баг или контракт?

**47.** `x = float('nan'); s = {x}; x in s` → True. `float('nan') in s` → False. Теперь `s = {float('nan'), float('nan')}` → `len(s)`? Объясни каждый из трёх результатов через одну и ту же строку C-кода.

**48.** У тебя 500M записей, каждая — dataclass из 8 полей, нужен дедуп по 3 из них. RAM 16GB. Сгенерированный `__hash__` от `tuple` полей + `set` не влезает. Какая архитектура? Что даст замена `__hash__` на 64-битный отпечаток и какова цена ложных срабатываний?

**49.** `dataclasses` при `eq=True, frozen=False` явно пишет `__hash__ = None`, хотя мог бы оставить наследованный. Какой класс ошибок это предотвращает и почему `attrs` исторически решил иначе? Кто прав?

**50.** `Decimal('1.0') == 1.0` → True, но `Decimal('0.1') == 0.1` → False. При этом контракт требует равных хешей у равных чисел. Как `_Py_HashDouble` и `Decimal.__hash__` приводятся к одному значению? Что за модуль `_PyHASH_MODULUS` и почему он — простое число?

**51.** Класс кеширует хеш: `self._h = hash(self._data)` в `__init__`. Через год кто-то добавил `__setattr__` для ленивой загрузки поля. Опиши, при каком конкретном порядке операций кешированный хеш станет неверным, и почему баг проявится только на dict размером > 5 элементов.

**52.** Байткод: `dis.dis('a < b < c')` vs `dis.dis('a < b and b < c')`. Какая инструкция появляется только в первом? Что делает `SWAP`/`COPY` вокруг неё и почему без цепочки Python не может использовать ту же оптимизацию?

**53.** Сравнение строк — timing-атака. Почему `secrets.compare_digest` существует, если `str.__eq__` в CPython начинается с проверки длины и `memcmp`? Что именно утекает и почему проверка длины — уже утечка?

**54.** `sorted(objects, key=attrgetter('x'))` на 10M элементов: 4.2s. `sorted(objects)` с `__lt__` по тому же полю: 31s. Разница в 7 раз при одинаковом числе сравнений. Откуда она и что такое decorate-sort-undecorate на уровне C?

**55.** `total_ordering` проверяет `getattr(cls, op, None) is not getattr(object, op, None)`. Класс наследуется от базы, где `__lt__` определён, но нужен другой порядок в подклассе. Что произойдёт и как это диагностировать, если ошибка тихая?

**56.** `functools.cmp_to_key` возвращает C-реализацию `_functools.cmp_to_key` при наличии. Почему Python-реализация в 2-3 раза медленнее при одинаковом числе вызовов `cmp`? Что там аллоцируется на каждое сравнение?

**57.** Проектируешь публичную библиотеку с типом `Money(amount, currency)`. Trade-off: `Money(1, 'USD') < Money(1, 'EUR')` — исключение, `NotImplemented` или сравнение по amount? Каждый вариант ломает какой-то реальный use-case. Твой выбор и что ты приносишь в жертву?
