# 20 ВОПРОСОВ НА СЕНЬОРНОСТЬ

**37.** PEP 328 разрешил относительные импорты и в 3.0 убил неявные относительные. Какой конкретный класс багов из py2 это лечило и какую цену заплатили большие пакеты?

**38.** `sys.modules['foo'] = None` — что произойдёт при следующем `import foo`? Откуда взялась эта семантика и кто ей пользуется?

**39.** До 3.3 модуль в `sys.modules` при `from a import b` резолвился только через `getattr`. В 3.7 (bpo-30024) добавили fallback на повторный `sys.modules`. Какой цикл это починило и почему не починило все?

**40.** Import lock: до 3.3 был один глобальный, стал per-module. Опиши deadlock, который возможен и сегодня, с двумя потоками и двумя модулями. Что делает `_bootstrap._blocking_on`?

**41.** `python -X importtime` показывает 400 мс на `import pandas`. Что конкретно дорого — I/O стата по `sys.path`, unmarshal `.pyc` или выполнение top-level кода? Как разделить эти три вклада измерением?

**42.** Каждая запись в `sys.path` при промахе — это N системных вызовов. Что такое directory cache в `FileFinder` и по какому признаку он инвалидируется? Почему mtime-гранулярность в 1 секунду тут опасна?

**43.** `pickle` сериализует функцию как `module.qualname`. Как ломается unpickle, если объект был запикшен из модуля, запущенного как `__main__`? Твоя архитектура для multiprocessing на Windows (spawn)?

**44.** PEP 552 (hash-based pyc) — зачем, если timestamp работал? Что такое `check_source` флаг, почему `unchecked` быстрее и какой это создаёт вектор атаки при shared `__pycache__`?

**45.** Импорт исполняет произвольный код на уровне модуля. Опиши атаку через `sys.path[0]` в скрипте, запущенном из writable-директории. Чем PEP 582 / `-P` / `PYTHONSAFEPATH` это меняют?

**46.** В Go импорт — статический граф, циклы = ошибка компиляции. В Python цикл иногда работает. Почему Python не может запретить циклы? Что фундаментально мешает — динамика или дизайн?

**47.** Namespace packages (PEP 420) делают `_NamespacePath` вместо list. Почему нельзя было обойтись списком? Что происходит при мутации `sys.path` после того, как namespace package уже импортирован?

**48.** У тебя монорепа: 200 пакетов, cold start CLI 2.8 с, из них 2.1 с — импорты. Бюджет — 300 мс. Твоя стратегия? Разбери lazy import, `__getattr__` на модулях, deferred `TYPE_CHECKING`, zipimport — что даст сколько и что сломает.

**49.** PEP 562 добавил `__getattr__` и `__dir__` на модулях. Почему это не сделали через подмену класса модуля на подкласс `ModuleType` (как делали хаком до 3.7)? Что было не так с тем хаком?

**50.** `from mypkg import *` при наличии `__all__` с несуществующим именем — когда узнаешь об ошибке? Сравни с поведением `__all__` в `__init__.py` пакета, где имена — подмодули.

**51.** Что происходит с модулем при завершении интерпретатора? Почему в `__del__` глобальные имена внезапно `None` и что изменил PEP 442 / bpo-39026 в этой части?

**52.** `importlib` в 3.3 переписали с C на Python и вморозили байткод в бинарник (`_frozen_importlib`). Зачем frozen, а не просто оставить C? Как теперь bootstrap-ится сам `importlib` до того, как работают импорты?

**53.** `zipimport` не поддерживает `.so` внутри архива. Почему это принципиально невозможно на Linux без распаковки? Как это обходят wheel-based tools и чего это стоит?

**54.** Ты пишешь плагин-систему: сторонние пакеты регистрируются через entry points. Сравни `importlib.metadata.entry_points()` с `pkg_resources` по времени старта и по семантике. Где ломается при editable install?

**55.** `sys.modules` — обычный dict, любой код может его мутировать. Тест A мокает `sys.modules['redis']`, тест B падает через 200 тестов. Как ты это диагностируешь и какую изоляцию строишь в CI?

**56.** Circular import между `models.py` и `services.py` починили через локальный импорт внутри функции. Какая цена в рантайме за каждый вызов? Что делает `IMPORT_NAME` при уже загруженном модуле и почему это не бесплатно?
