# 20 ВОПРОСОВ НА СЕНЬОРНОСТЬ

**39.** CPython хранит `int` как массив digit по 30 бит на 64-битной платформе (`PYLONG_BITS_IN_DIGIT`). Почему 30, а не 32? Что даёт запас в 2 бита при сложении и умножении?

**40.** До Python 3.12 `PyLongObject` использовал `ob_size` со знаком для хранения и знака числа, и количества digits. В 3.12 (PEP не нужен, gh-101291) это заменили на тегированное поле `lv_tag`. Что дало изменение и почему нельзя было оставить как было?

**41.** Python 3.11 ввёл лимит на конвертацию `int` ↔ `str` — 4300 цифр по умолчанию (`sys.set_int_max_str_digits`). Какая CVE и какой класс DoS это чинит? Почему алгоритм `str(int)` квадратичный и почему его просто не заменили на быстрый?

**42.** `NSMALLPOSINTS = 257`, `NSMALLNEGINTS = 5` — small int cache даёт `[-5, 256]`. Почему нижняя граница именно -5, а верхняя 256? Что случится с производительностью, если расширить кеш до `[-10000, 10000]`?

**43.** В Python 3.12 small ints стали immortal (PEP 683). Как это связано с per-interpreter GIL (PEP 684) и что было бы с refcount на закешированной единице при 32 потоках без immortality?

**44.** `sum([0.1] * 10) == 1.0` → `False`, но `math.fsum([0.1] * 10) == 1.0` → `True`. Какой алгоритм внутри `fsum`, какая у него сложность по памяти и почему `sum` не переписали на него?

**45.** `hash(2**61 - 1) == hash(0)` в CPython. Почему? Как устроен `_PyHASH_MODULUS` и какой класс атаки на `dict` из этого следует — и почему PYTHONHASHSEED тут не спасает?

**46.** `x = 2**53; x + 1.0 == x` → `True`. При этом `x + 1 == x` → `False`. Объясни, в какой момент происходит потеря и почему сравнение `int` с `float` в CPython НЕ работает через приведение int к float. Где это в `float_richcompare`?

**47.** `Decimal('1.0') == Decimal('1.00')` → `True`, но `hash` у них равен, а `repr` разный, и `Decimal('1.0') is not Decimal('1.00')`. Что такое cohort в стандарте IEEE 754-2008 decimal и какое инвариантное правило Python обязан соблюдать для `==` и `hash`?

**48.** На CPU без `_decimal` (чистый Python `_pydecimal`) `Decimal` медленнее float в ~100 раз, с libmpdec — в ~10-20. Где именно libmpdec выигрывает и почему `Decimal` всё равно не заменит float в численных методах даже при равной скорости?

**49.** `Fraction` арифметика вызывает `math.gcd` на каждой операции. Покажи, где это даст экспоненциальный blow-up: сложение 1000 дробей подряд. Как выглядит правильная архитектура суммирования дробей и какой алгоритм сокращения используется в 3.11+ (`_normalize`)?

**50.** В Java `int` переполняется молча (wrap-around), в Rust — паникует в debug и wrap в release, в Python — растёт. Какую цену Python платит на горячем цикле `for i in range(10**8): s += i`? Что делает адаптивный интерпретатор 3.11+ с `BINARY_OP` для int (PEP 659) и когда специализация деоптимизируется?

**51.** `math.floor(2.675 * 100) == 267`, а не 268. Объясни цепочку от парсинга литерала до результата. Почему `Decimal('2.675') * 100` даёт другое и где начинающие делают `round(x, 2)` в деньгах и теряют копейки при агрегации миллиона строк?

**52.** `float.__repr__` с Python 3.1 использует алгоритм David Gay / Grisu-подобный кратчайшего представления вместо `%.17g`. Что было в py2 и почему `repr(0.1)` печатал `0.10000000000000001`? Какой инвариант гарантирует новый алгоритм и какой НЕ гарантирует?

**53.** У тебя 500M денежных транзакций, нужно точное суммирование, RAM 16GB. `Decimal` — 104 байта на объект, `int` в копейках — 28. Твоя архитектура? Где сломается int-в-копейках и что делать с курсами валют и делением при сплите?

**56.** `nan` нарушает рефлексивность: `x != x`. При этом `x = float('nan'); x in [x]` → `True`. Объясни, как `list.__contains__` и `dict` обходят это. Какое правило `PyObject_RichCompare` тут срабатывает и почему это осознанное нарушение контракта `__eq__`?
