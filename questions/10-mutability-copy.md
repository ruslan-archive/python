# 20 ВОПРОСОВ НА СЕНЬОРНОСТЬ

**33.** `copy.deepcopy(x)` в 3.12 в разы медленнее наивного `pickle.loads(pickle.dumps(x))` на больших однородных структурах. Почему — и в каком случае pickle-трюк даёт неверный результат?

**34.** `memo` в `deepcopy` — это `dict`, где ключ = `id(obj)`. Что мешает объекту быть собранным GC во время копирования и что делает `memo[id(memo)] = [...]` (`_keep_alive`) в CPython?

**35.** PEP 671 (late-bound defaults, `def f(x=>[])`) отклонён/отложен. Какую именно проблему он решал, какой ценой для интроспекции `__defaults__` и почему сообщество не приняло?

**36.** `def f(a=[])` — где физически живёт этот список? Покажи через `f.__defaults__` и объясни, как он попадает в объект функции на этапе `MAKE_FUNCTION`, а не при вызове.

**37.** Ruff/flake8-bugbear ловят B006 (mutable default) статически. Почему они пропускают `def f(a=SOME_CONST)`, где `SOME_CONST = []` на уровне модуля? Это false negative или сознательный trade-off?

**38.** `str` иммутабелен, но CPython делает in-place `+=` для строк при refcount == 1 (`unicode_concatenate` в `ceval`). Как это сочетается с иммутабельностью и что ломается, если добавить `sys.getrefcount`-наблюдателя?

**39.** Кортеж хеширует свои элементы при вызове `hash()`, а не при создании. Как из-за этого мутабельный элемент внутри «хешируемого» кортежа роняет `dict` в inconsistent state — и почему это не считают багом?

**40.** В Rust мутабельность = свойство binding'а (`let mut`), проверяется компилятором. В Python — свойство типа, проверяется в рантайме. Какой класс багов Rust ловит бесплатно, а Python не может поймать даже с mypy? А что мешает mypy?

**41.** `copy` module диспатчит через `_copy_dispatch` — plain `dict` от типа к функции. Что происходит с подклассом `list` без `__copy__`: попадёт ли он в fast path? Покажи, где именно `cls = type(x)` рушит наивное ожидание.

**42.** Ты пишешь библиотеку. `deepcopy` объекта твоего класса должен вернуть `self` (объект логически иммутабелен). Три способа: `__deepcopy__`, `__reduce__`, `copyreg.pickle`. Что выберешь и какой сюрприз ждёт пользователя при `pickle` в каждом варианте?

**43.** `deepcopy` рекурсивен → на структуре глубиной 10к падает с `RecursionError`. Твоя архитектура итеративного deepcopy: как хранить стек, как обрабатывать `memo`, как не потерять порядок восстановления для объектов с `__setstate__`?

**44.** Кеш на 500M небольших `dict`, RAM 16GB. Отдавать наружу нужно защитно (клиент не должен мутировать кеш). `deepcopy` на каждом чтении убивает перф. Твоя архитектура? Оцени, что даст `MappingProxyType`, а что нет.

**45.** `types.MappingProxyType(d)` — read-only view, не копия. Какой конкретный вектор атаки/бага он НЕ закрывает и почему его не считают security-механизмом? Как достать оригинальный `dict` из proxy?

**46.** Десериализация недоверенного ввода через `copy.deepcopy` объекта, чей класс определяет `__reduce__`. Возможен ли RCE через `copy`, а не через `pickle`? Обоснуй, глядя на `_reconstruct`.

**47.** До Python 3.7 `dict` копировался с потерей порядка вставки в некоторых путях, после — порядок часть контракта (PEP 468/языковая гарантия с 3.7). Что это поменяло в семантике `dict.copy()` и какой класс тестов внезапно стал зелёным/красным?

**48.** `list(x)`, `x[:]`, `copy.copy(x)`, `x.copy()` для списка из 1M элементов дают разный тайминг. Расставь по возрастанию времени и объясни разницу через байткод (`dis`) и `list_slice`/`PyList_GetSlice`.

**49.** `a = [1,2,3]; b = a; a += [4]` мутирует `b`, а `a = a + [4]` — нет. Покажи через `dis`, какие опкоды различаются (`INPLACE_ADD`/`BINARY_OP` с NB_INPLACE) и почему для `tuple` разницы нет.

**50.** Two threads делают `d.copy()` и `d[k] = v` на одном `dict` без блокировки. Атомарна ли `dict.copy()` в CPython 3.12? А в 3.13 free-threaded (PEP 703)? Что поменялось и какой ценой.

**51.** Дизайн-вопрос без правильного ответа: API возвращает внутренний `list`. Варианты — вернуть as-is (документировав «не мутируй»), вернуть копию, вернуть `tuple`, вернуть proxy. Что выберешь для библиотеки на 10k пользователей и что для internal-сервиса? Объясни разницу в критериях.
