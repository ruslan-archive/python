# 20 ВОПРОСОВ НА СЕНЬОРНОСТЬ

**35.** Arena в pymalloc = 256 KB (с 3.10; было 1 MB как mmap-регион в некоторых сборках). Почему CPython держит arenas в usable-списке, отсортированном по числу свободных pools, а не берёт первую попавшуюся?

**36.** `Py_INCREF`/`Py_DECREF` — не атомарные операции. В Java refcount так не сделали бы. Почему CPython платит GIL за право иметь неатомарный inc/dec, и что делает free-threaded build (PEP 703) вместо этого — что такое biased reference counting?

**37.** Immortal objects (PEP 683, 3.12): `None`, `True`, малые int получили refcount `_Py_IMMORTAL_REFCNT`. Какую конкретную проблему это решило для fork+CoW и для nogil? Какую цену заплатили?

**38.** Ты форкнул воркеры gunicorn. RSS каждого воркера растёт, хотя код read-only. Объясни механизм через refcount и CoW. Как `gc.freeze()` перед fork меняет картину и почему только частично?

**39.** Алгоритм mark & sweep в CPython не mark & sweep, а «subtract internal references». Опиши, зачем нужен второй проход `move_unreachable` и почему нельзя просто считать объект с `gc_refs == 0` мусором.

**40.** До 3.4 объекты с `__del__`, попавшие в цикл, уходили в `gc.garbage` и не собирались. PEP 442 это починил. Как именно? Почему решение безопасно при resurrection, но `__del__` теперь может вызваться на объекте с уже мёртвыми полями?

**41.** `sys.getsizeof(1)` = 28, `sys.getsizeof(2**30)` = 32, `sys.getsizeof(2**60)` = 36. Объясни шаг 4 байта и почему в 3.12 внутреннее представление `int` поменяли (`_PyLong_IsCompact`). Где выгода?

**42.** У тебя 500M коротких строк-ключей в одном процессе, RAM 16 GB. Наивный `dict[str, int]` не влезает. Опиши свою архитектуру: что режешь — оверхед `PyObject`, оверхед dict, или сам Python? Обоснуй числами.

**43.** Список из 10M элементов: `list.append` в цикле vs `[None]*10_000_000` с присвоением по индексу. Разница по памяти и по времени. Объясни через growth factor `list_resize` (какой он в CPython?) и `realloc`.

**44.** Memory fragmentation: процесс освободил 8 GB объектов, RSS не упал. Ты знаешь про arenas. При каком паттерне аллокаций arena не вернётся в ОС, даже если в ней занят один блок? Как это чинить, не переписывая код?

**45.** `PYTHONMALLOCSTATS=1` показывает большое число `arenas highwater mark` при малом текущем. Что произошло и почему pymalloc не отдал память? В чём разница между «свободный pool» и «arena, готовая к munmap»?

**46.** GC generational hypothesis взята из Java/Lisp. Но CPython не перемещает объекты (non-moving) и не делает bump allocation. Что из плюсов generational GC при этом теряется? Почему CPython не может стать moving?

**47.** `gc.set_threshold(0)` отключает GC. Ты видишь -15% latency p99 в сервисе. При каком классе объектов это безопасно, а при каком через сутки прилетит OOM? Как проверить, не выключая на проде?

**48.** DoS-угол: атакующий шлёт JSON, который после парсинга образует граф из 10M мелких объектов с циклами. Что происходит с GC pause? Как `gc.freeze()` + отключение gen2 меняет вектор атаки?

**49.** `__slots__` убирает `__dict__`, но не всегда даёт выигрыш. Назови три случая, когда `__slots__` не сработает или сработает во вред (подсказка: наследование, weakref, `__dict__` в базе).

**50.** Key-sharing dictionaries (PEP 412): 1M инстансов класса с 5 атрибутами. Сколько экономит shared key table и при каком действии в рантайме она «расшаривается обратно» (unshare), убив всю экономию?

**51.** В Rust нет GC — RAII + borrow checker. В Go — concurrent tri-color mark&sweep без refcount. CPython выбрал refcount + backup GC. Назови конкретное свойство CPython API (какое именно?), которое делает переход на чистый tracing GC практически невозможным.

**52.** Утечка в проде: RSS растёт 200 MB/день, `gc.get_objects()` не показывает роста Python-объектов, `tracemalloc` чист. Где искать? Назови три места, которые оба инструмента не видят.

**53.** `id()` в CPython = адрес. Объект удалён → адрес переиспользован → `id(a) == id(b)` для разных объектов в разное время. Приведи реальный баг, который так возникает (кеш по `id`?), и security-угол: что `id()` утекает про layout процесса?

**54.** Trade-off без правильного ответа: сервис на 40 GB heap. Вариант A — `gc.disable()` + рестарт воркера каждые N запросов. Вариант B — тюнинг thresholds `(100_000, 50, 50)`. Вариант C — вынести данные в `mmap`/Arrow вне heap. Что выбираешь и что теряешь?
